HW #6 : uC/OS-III 태스크 간 통신 과제
HW #6 과제내용
3개 이상의 task로 구성된 시스템을 만들어, 인터럽트(ISR) 기반 입력 -> task 간 통신 -> 출력 흐름을 구현하고, 동작 상태를 USART로 모니터링한다.

필수 요구사항
3개 이상의 task를 생성, 각 task는 서로 다른 우선순위를 가지며 입력 task / 출력 task / 모니터링 task로 역할을 분리한다.
입력은 반드시 인터럽트(ISR) 기반으로 처리한다.
ISR에서는 인터럽트 플래그 클리어 + OSSemPost() 또는 OSQPost() 만 수행
실제 처리(파싱·판단·출력)는 입력 task에서 수행
폴링(while에서 GPIO 반복 read)만으로 입력을 처리하면 큰 폭 감점
task 간 통신은 세마포어 또는 메시지 큐를 사용한다 (전역 변수만으로 처리 불가).
USART로 입력 인식 / 출력 처리 상태 / 이벤트 전달이 확인 가능해야 한다.
입력 / 출력 장치
입력: 보드 사용자 버튼, 외부 버튼, 외부 센서의 디지털 출력 중 1개 이상
출력: LED1·LED2·LED3, 외부 LED, 부저, 모터·서보 등 1개 이상 (기본은 보드 LED)
예시 주제 (자유 선택)
순환 LED 정지 게임 - LED 순환 점멸 -> 버튼 입력 시 정지, USART로 지정한 목표 LED와 일치 여부 확인
버튼 입력 패턴 기반 LED 모드 전환 - 짧게/길게/두 번 누름으로 LED 모드 전환
USART 명령 + 버튼 혼합 제어 - USART로 모드 지정, 버튼으로 실행·단계 전환
예시 또는 동일한 요구사항을 만족하는 자유 주제 가능.
